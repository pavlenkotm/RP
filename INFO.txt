📋 НАЗНАЧЕНИЕ
─────────────────────────────────────────────────────────────────────────────
Программа предназначена для автоматического формирования документов 
"Расчёт на прочность" для детского игрового оборудования.

✨ ОСНОВНЫЕ ВОЗМОЖНОСТИ
─────────────────────────────────────────────────────────────────────────────
✓ Чтение данных из Excel каталога изделий
✓ Парсинг PDF паспортов с извлечением технических характеристик
✓ Автоматическая генерация Word документов по шаблону
✓ Вставка изображений с автоматическим масштабированием
✓ Расчёт нагрузок (дети, снег, ветер) для Санкт-Петербурга
✓ Поддержка 6 категорий изделий
✓ Подробное логирование всех операций
✓ Обработка до 300 изделий за один запуск

💻 СИСТЕМНЫЕ ТРЕБОВАНИЯ
─────────────────────────────────────────────────────────────────────────────
• Операционная система: Windows 10/11
• Python: 3.8 или выше
• Microsoft Word: 2016 или выше (для обновления полей)
• Свободное место: ~100 МБ
• Оперативная память: 4 ГБ (рекомендуется 8 ГБ)

📥 УСТАНОВКА
─────────────────────────────────────────────────────────────────────────────
1. Установите Python 3.8+ с https://www.python.org/downloads/
   ⚠️ При установке отметьте "Add Python to PATH"

2. Запустите deploy.bat для установки зависимостей

3. Настройте config/config.json (укажите пути к файлам)

4. Готово к работе!

🚀 БЫСТРЫЙ СТАРТ
─────────────────────────────────────────────────────────────────────────────
1. Запустите run.bat

2. Выберите "Тестовый запуск" для проверки на одном изделии

3. Если тест прошёл успешно, выберите "Полная обработка"

4. Результаты будут сохранены в папку, указанную в конфигурации

📊 ИСХОДНЫЕ ДАННЫЕ
─────────────────────────────────────────────────────────────────────────────
Для работы программы требуются:

1. Excel файл с каталогом изделий
   Столбцы: A (артикул), B (наименование), D (путь к картинке), 
            E (количество детей)

2. Паспорта изделий (PDF)
   Формат имени: должен содержать артикул изделия

3. Шаблон Word
   Файл с рамками, штампами и структурой расчёта

4. Изображения общего вида изделий

📂 ВЫХОДНЫЕ ДАННЫЕ
─────────────────────────────────────────────────────────────────────────────
• Word документы: артикул_наименование_РП.docx
• Лог-файл: log_РП.txt (все операции и ошибки)

⚡ ПРОИЗВОДИТЕЛЬНОСТЬ
─────────────────────────────────────────────────────────────────────────────
• Одно изделие: ~2 секунды
• 100 изделий: ~3-5 минут
• 300 изделий: ~10-15 минут

🐛 РЕШЕНИЕ ПРОБЛЕМ
─────────────────────────────────────────────────────────────────────────────
Проблема: "Python не распознан"
Решение: Переустановите Python с опцией "Add to PATH"

Проблема: "Excel файл не найден"
Решение: Проверьте путь в config.json, используйте двойные слэши: D:\\папка

Проблема: "Паспорт не найден"
Решение: Убедитесь, что имя PDF содержит артикул изделия

Проблема: "Поля не обновляются"
Решение: Убедитесь, что установлен Microsoft Word
